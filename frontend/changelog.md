### Changelog

#### Frontend
- **Архитектура и Рефакторинг**:
    - Переход на структуру с использованием `srcDir: 'app/'` для лучшей организации кода.
    - Внедрены специализированные композаблы: `useApi` для централизованной работы с HTTP-запросами и `useProjects` для управления бизнес-логикой проектов.
    - Выполнена декомпозиция страницы редактирования проекта (`[id].vue`) на независимые функциональные компоненты: `ProjectScriptStep`, `ProjectAudioStep`, `ProjectVideoStep`.
    - Добавлен переиспользуемый UI-компонент `AppConfirmModal` для подтверждения деструктивных действий.
- **Оптимизация и UX**:
    - Реализовано локальное кеширование проектов в `localStorage` для мгновенного отображения данных в Dashboard до завершения сетевых запросов.
    - Улучшена логика переключения шагов генерации (Script -> Audio -> Video) с поддержкой состояний очереди и фоновой обработки.
    - Настроена автоматическая вставка токенов авторизации и обработка ошибок 401 (автоматический разлогин).
- **Типизация и Исправления**:
    - Внедрена строгая типизация TypeScript для всех сущностей (`Project`, `Status`, `User`).
    - Исправлены алиасы путей (`~` и `@`) в `nuxt.config.ts`, устраняющие ошибки импорта при использовании `srcDir`.
    - Оптимизирована работа со статусами проектов (переход на числовые значения для предотвращения конфликтов типов в шаблонах).

#### Backend
- **Интеграции**:
    - Добавлена поддержка **ElevenLabs** и **GenAiPro** для высококачественной генерации голоса.
    - Реализована интеграция с альтернативными провайдерами озвучки через общие интерфейсы.
- **Асинхронность и Очереди**:
    - Внедрен **MassTransit** с использованием RabbitMQ для фоновой генерации аудио и видео.
    - Добавлен потребитель `GenerateAudioConsumer` для обработки длительных задач синтеза речи.
- **Инфраструктура**:
    - Улучшена логика ожидания (polling) готовности файлов во внешних сервисах.
    - Расширена конфигурация провайдеров (ApiKey, BaseUrl, модели голосов) в настройках приложения.
    - Скорректирована регистрация зависимостей (DI) для поддержки множественных стратегий генерации.
